===========

BEFORE (manual startup in 4 terminals):
  Terminal 1: docker compose up -d
  Terminal 2: npm run dev (frontend dev server)
  Terminal 3: uvicorn api.main:app --reload
  Terminal 4: cd consumer && python pipeline.py

AFTER (automatic startup in 3 terminals):
  Terminal 1: docker compose up
  Terminal 2: uvicorn api.main:app --reload
  Terminal 3: cd consumer && python pipeline.py

Result: http://localhost:3000 opens automatically with production build!

DOCKER COMMANDS REFERENCE:
==========================

docker compose up              → Start everything
docker compose up -d           → Start in background
docker compose down            → Stop everything
docker compose down -v         → Stop + reset Redis
docker compose logs -f         → Watch all logs
docker compose logs -f frontend→ Watch frontend only
docker compose ps              → Show service status
docker compose up --build      → Rebuild frontend if code changed


FAQ:
====

Q: Will the frontend auto-rebuild if I change code?
A: No, Docker runs a production build. For development:
   - docker compose stop frontend
   - cd frontend && npm run dev
   
Q: What if port 3000 is already in use?
A: Either:
   - Kill the process with lsof -t -i :3000
   - Or change docker-compose.yml "3000:3000" to "3001:3000"

Q: Can I still run frontend dev mode?
A: Yes! Just stop Docker frontend and run npm run dev locally

Q: Does Docker need the built dist/ folder?
A: No, Docker builds it from source. It's in .gitignore

Q: How big is the Docker image?
A: ~500MB (includes Node.js 18 base, npm install, build artifacts)
